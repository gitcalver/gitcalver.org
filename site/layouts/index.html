Deterministic version numbers from git history

gitcalver derives unique, strictly increasing version numbers from your commit timestamps. No manual bumps. No config files. Just git.

How it works

Every first-parent commit on your default branch gets a version based on its UTC commit date and position within that day:

CommitDate (UTC)Version
a1b2c3d{{ $yesterdayISO }} 09:00{{ $yesterday }}.1
e4f5a6b{{ $yesterdayISO }} 14:30{{ $yesterday }}.2
c7d8e9f{{ $yesterdayISO }} 17:00{{ $yesterday }}.3
0a1b2c3{{ $todayISO }} 10:00{{ $today }}.1

Three commits on the same day → {{ $yesterday }}.1, .2, .3. New day resets the count.
